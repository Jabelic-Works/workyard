# Catalog

The catalog is agent-readable engineering guidance.

It should explain how work should be done across stacks, layers, patterns, and verification checks without relying on private context.

## Sections

- `stacks/`: stack-level guidance such as TypeScript libraries, web apps, APIs, or documentation sites
- `layers/`: cross-stack layers such as frontend, backend, database, API contracts, testing, and release
- `patterns/`: reusable implementation patterns
- `checks/`: verification expectations before edits, commits, pull requests, or releases
